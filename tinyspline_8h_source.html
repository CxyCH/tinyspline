1043 /******************************************************************************
1044 * *
1045 * :: Transformation functions *
1046 * *
1047 * TinySpline is a library focusing on transformations. That is, most *
1048 * functions are used to transform splines by modifying their state, e.g., *
1049 * their number of control points, their degree, and so on. Accordingly, each *
1050 * transformation functions specifies an input and output parameter (along *
1051 * with the other parameters required to calculate the actual transformation). *
1052 * By passing a different pointer to the output parameter, the transformation *
1053 * result is calculated and stored without changing the state of the input. *
1054 * This is in particular useful when dealing with errors as the original state *
1055 * will never be modified. For instance, let's have a look at the following *
1056 * code snippet: *
1057 * *
1058 * tsBSpline in = ... // an arbitrary spline *
1059 * tsBSpline out; // result of transformation *
1060 * *
1061 * // Subdivide 'in' into sequence of bezier curves and store the result *
1062 * // in 'out'. Does not change 'in' in any way. *
1063 * tsError err = ts_bspline_to_beziers(&in, &out); *
1064 * if (err != TS_SUCCESS) { *
1065 * // fortunately, 'in' has not been changed *
1066 * } *
1067 * *
1068 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
1069 * changed allowing you to handle the error properly. *
1070 * *
1071 * Unless stated otherwise, the order of the parameters for transformation *
1072 * functions is: *
1073 * *
1074 * function(input, [additional_input], output, [additional_output]) *
1075 * *
1076 * 'additional_input' are parameters required to calculate the actual *
1077 * transformation. 'additional_output' are parameters storing further result. *
1078 * *
1079 * Note: None of TinySpline's transformation functions frees the memory of the *
1080 * output parameter. Thus, when using the same output parameter multiple *
1081 * times, make sure to free memory before each call. Otherwise, you will *
1082 * have a bad time with memory leaks: *
1083 * *
1084 * tsBSpline in = ... // an arbitrary spline *
1085 * tsBSpline out; // result of transformations *
1086 * *
1087 * ts_bspline_to_beziers(&in, &out); // first transformation *
1088 * ... // some code *
1089 * ts_bspline_free(&out); // avoid memory leak. *
1090 * ts_bspline_buckle(&in, &out); // next transformation *
1091 * *
1092 * If you want to modify your input directly without having a separate output, *
1093 * pass it as input and output at once: *
1094 * *
1095 * tsBSpline s = ... // an arbitrary spline *
1096 * tsReal *knots = ... // a knot vector *
1097 * *
1098 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
1099 * *
1100 * Note: If a transformation function fails *and* input != output, all fields *
1101 * of the output parameter are set to 0/NULL. If input == output, your *
1102 * input may have an invalid state in case of errors. *
1103 * *
1104 ******************************************************************************/
